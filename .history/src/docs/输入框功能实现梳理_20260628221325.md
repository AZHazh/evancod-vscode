- **输入框主体**：`ChatInput` 负责 textarea、附件、`@` 文件搜索、`/` 命令菜单、权限、上下文用量、模型选择和发送逻辑，入口在 `desktop/src/components/chat/ChatInput.tsx:89`。
- **目录/文件添加**：`+` 菜单调用 `openAttachmentPicker`，桌面端走原生文件选择，浏览器退回 `<input type=file>`；`@` 触发 `FileSearchMenu`，通过 `/api/filesystem/browse` 浏览/搜索文件与目录，并保存为 workspace reference。关键代码在 `desktop/src/components/chat/ChatInput.tsx:921`、`desktop/src/components/chat/ChatInput.tsx:1038`、`desktop/src/components/chat/FileSearchMenu.tsx:26`、`src/server/api/filesystem.ts:130`。
- **`/` 命令**：`findSlashTrigger` 只在光标前出现独立 `/` 且 filter 无空白/换行时打开菜单；命令来源是服务端 session slash commands、技能命令、桌面 fallback、本地 agent 命令合并。命令清单含 `agent/mcp/skills/help/status/cost/context/plugin/memory/doctor/compact/clear/goal/review/commit/pr/init/bug/config/login/logout/model/permissions/terminal-setup/vim`，定义在 `desktop/src/components/chat/composerUtils.ts:58`，触发/替换逻辑在 `desktop/src/components/chat/composerUtils.ts:268`。
- **本地弹窗命令**：`mcp/skills/help/status/cost/context` 不直接发给模型，而是在提交时打开 `LocalSlashCommandPanel`；`config/plugin/memory/doctor` 会打开设置页；其他命令按普通消息发给 CLI。见 `desktop/src/components/chat/ChatInput.tsx:663`、`desktop/src/components/chat/LocalSlashCommandPanel.tsx:19`。
- **访问权限**：工具执行权限模式由 `PermissionModeSelector` 切换，支持 `default/acceptEdits/plan/bypassPermissions`；危险的 `bypassPermissions` 先弹确认。切换会通过 WebSocket 发 `set_permission_mode`，服务端转成 SDK control request，并持久化到 session metadata。见 `desktop/src/components/controls/PermissionModeSelector.tsx:31`、`desktop/src/stores/chatStore.ts:1082`、`src/server/ws/handler.ts:226`、`src/server/services/conversationService.ts:504`。
- **工具授权弹窗链路**：CLI 发 `can_use_tool` control request，服务端翻译为 `permission_request`，前端存入 `pendingPermission` 并由权限弹窗响应；响应经 `permission_response` 回到 SDK，允许时可带 rule/updatedInput/updatedPermissions，拒绝时带 deny message。见 `src/server/ws/handler.ts:1731`、`desktop/src/stores/chatStore.ts:1048`、`src/server/services/conversationService.ts:462`。
- **上下文弹窗/内容**：右侧圆环是 `ContextUsageIndicator`，定时或会话变化时调用 `sessionsApi.getInspection(...includeContext)`，展示 context token、模型、分类明细；`/context` 则打开 `LocalSlashCommandPanel` 的 inspector context tab。见 `desktop/src/components/chat/ContextUsageIndicator.tsx:72`、`desktop/src/api/sessions.ts:373`、`src/server/api/sessions.ts:540`。
- **模型与推理强度**：`ModelSelector` 合并官方模型、OpenAI 官方模型和自定义 provider 模型；会话级选择写入 `useSessionRuntimeStore` 的 `localStorage`，并通过 `set_runtime_config` 发给服务端。推理强度为 `low/medium/high/max`，作为 `effortLevel` 保存并在 CLI 重启/启动时传入 runtime settings。见 `desktop/src/components/controls/ModelSelector.tsx:165`、`desktop/src/components/controls/ModelSelector.tsx:500`、`desktop/src/stores/sessionRuntimeStore.ts:4`、`src/server/ws/handler.ts:678`、`src/server/ws/handler.ts:2446`。
