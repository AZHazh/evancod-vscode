## 🚧 五、风险与应对

### 风险 1: VSCode API 限制
**风险**: Terminal API 无法获取命令输出
**应对**: 使用隐藏终端 + 临时文件方案

### 风险 2: 复制代码兼容性
**风险**: cc-desktop 代码使用 Bun 特性
**应对**: 提前识别不兼容代码，准备替代方案

### 风险 3: 性能问题
**风险**: Webview 性能不如原生 Electron
**应对**: 虚拟滚动、延迟加载、性能监控

### 风险 4: OAuth 回调复杂性
**风险**: 本地 HTTP 服务器在 VSCode 中可能有限制
**应对**: 使用 VSCode 自定义 URI Scheme 作为备选

### 风险 5: 时间延期
**风险**: 某些阶段可能超期
**应对**: 设置 Buffer 时间，优先级排序，可裁剪功能
