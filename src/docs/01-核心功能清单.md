# VSCode Evancod 核心功能清单

> 基于 cc-desktop-main 项目分析，明确 VSCode 插件版本必须具备的核心功能

## 📋 一、Agent 核心引擎（必须完整移植）

### 1.1 查询引擎 (QueryEngine)
**源文件**: `src/QueryEngine.ts`, `src/query.ts`

**核心能力**:
- 用户消息 → API 调用 → 工具执行的完整循环
- 流式响应处理
- 上下文管理和压缩
- 消息历史持久化
- Token 使用追踪
- 错误重试机制

**移植要求**: 完整复用，无需修改核心逻辑

---

### 1.2 工具系统 (Tools)
**源文件**: `src/Tool.ts`, `src/tools/`

**必须移植的工具** (优先级排序):

#### Tier 1 - 核心文件操作（必须）
1. **FileReadTool** - 读取文件
2. **FileEditTool** - 编辑文件（diff-based）
3. **FileWriteTool** - 写入新文件
4. **GlobTool** - 文件模式匹配
5. **GrepTool** - 内容搜索

#### Tier 2 - 执行环境（必须）
6. **BashTool** - 命令执行（需适配 VSCode Terminal API）
7. **NotebookEditTool** - Jupyter Notebook 编辑

#### Tier 3 - Agent 协作（必须）
8. **AgentTool** - 子 Agent 调用
9. **AskUserQuestionTool** - 用户交互
10. **TaskCreateTool** / **TaskGetTool** / **TaskUpdateTool** / **TaskListTool** - 任务管理

#### Tier 4 - 高级功能（必须）
11. **WebFetchTool** - 网页内容获取
12. **WebSearchTool** - 网络搜索
13. **MCPTool** - MCP 服务器调用
14. **SkillTool** - Skills 插件系统
15. **LSPTool** - 语言服务器协议

#### Tier 5 - 辅助工具（重要）
16. **EnterPlanModeTool** / **ExitPlanModeTool** - 计划模式
17. **EnterWorktreeTool** / **ExitWorktreeTool** - Worktree 隔离
18. **TodoWriteTool** - Todo 管理
19. **BriefTool** - 简报生成

**VSCode 适配要点**:
- BashTool 需要适配 `vscode.Terminal` API
- 文件操作需要适配 `vscode.workspace.fs`
- LSPTool 可以与 VSCode 原生 LSP 集成

---

## 📡 二、Provider 系统（核心基础设施）

### 2.1 多 Provider 支持
**源文件**: `src/services/api/`, `src/utils/model/providers.ts`

**支持的 Provider**:
- ✅ Anthropic Claude (firstParty)
- ✅ AWS Bedrock
- ✅ Google Vertex AI
- ✅ Azure OpenAI
- ✅ Palantir Foundry
- ✅ 自定义 OpenAI 兼容端点

**核心功能**:
- Provider 配置管理 (`~/.claude/cc-evancod/providers.json`)
- 动态切换 Provider
- 环境变量注入
- API 格式转换（Anthropic ↔ OpenAI）
- 网络代理支持

**移植要求**: 完整保留，VSCode 插件同样需要支持多 Provider

---

### 2.2 new-api 同步功能（必须）
**源文件**: `src/server/services/newApiSyncService.ts`, `newapi-sync-plan.md`

**核心流程**:
1. 用户输入 new-api 站点地址
2. OAuth 授权流程（浏览器 + 本地回调）
3. 获取 new-api 令牌列表
4. 按分组映射模型到 main/sonnet/opus/haiku 四槽
5. 导入为本地 Provider 配置

**技术要点**:
- 本地 HTTP 监听器接收 OAuth 回调
- 分组模型智能填充算法
- 完整 key 仅在后端处理，前端只展示脱敏版本

**移植策略**: 
- VSCode 插件同样需要本地 HTTP 服务（可以复用或简化）
- 授权流程调用系统浏览器
- 使用 VSCode 自定义 URI Scheme (`vscode://`)

## ❌ 不需要移植的功能

以下功能是 CLI/Desktop/IM 特定的，VSCode 插件**不需要**:

1. **Terminal UI** (`src/ink/`, `src/components/`) - 用 Webview 替代
2. **本地 API 服务器** (`src/server/`) - VSCode Extension Host 直接调用
3. **Electron 相关** (`desktop/electron/`) - 不适用
4. **IM 适配器** (`adapters/`) - VSCode 插件不需要
5. **CLI 命令** (`src/commands/`) - 部分逻辑可复用，但 UI 完全不同
6. **守护进程** (`src/daemon/`) - VSCode 自带进程管理
7. **SSH 远程** (`src/ssh/`) - VSCode 有原生 Remote 支持
