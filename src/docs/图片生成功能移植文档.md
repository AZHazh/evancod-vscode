# 生图功能完整解析与移植文档

> 源项目：`sub2api`（Go + Gin 后端 + Vue3/TS 前端）
> 目标项目：VSCode 插件（Vue3 + TS 的 Webview + Node 的 Extension Host）
> 入口：`frontend/src/components/admin/account/AccountTestModal.vue` 的 `@click="startTest"`

---

## 一、功能一句话概述

用户在弹窗里选一个模型、填一段提示词，点击"开始测试"。前端向后端发起一个 **POST** 请求，后端根据账号所属平台（Gemini / OpenAI）调用对应的**生图 API**，把返回的图片以 **base64 data URI** 的形式，通过 **SSE（Server-Sent Events）流式事件** 逐条推回前端。前端解析事件流，遇到 `image` 事件就把图片渲染到界面上。

核心要点：
1. **传输协议**：不是普通 JSON 响应，而是 SSE 流（`text/event-stream`），每条事件是一行 `data: {json}`。
2. **图片格式**：图片不是 URL 链接，而是 `data:image/png;base64,xxxx` 这种内联 data URI，前端 `<img :src>` 直接可用。
3. **生图判定**：靠"模型名"判断走不走生图路径（Gemini 看是否含 `-image`，OpenAI 看是否以 `gpt-image-` 开头）。
4. **两条上游路径**：Gemini 走流式 `streamGenerateContent`，OpenAI（API Key）走一次性的 `/v1/images/generations`。

### 3.2 是否显示"提示词输入框"与走生图逻辑的判定

前端用模型名 + 平台来决定是不是生图测试（决定是否显示提示词输入框）：

```ts
// Gemini 生图：模型名以 gemini- 开头且包含 -image
const supportsGeminiImageTest = computed(() => {
  const modelID = selectedModelId.value.toLowerCase()
  if (!modelID.startsWith('gemini-') || !modelID.includes('-image')) return false
  return props.account?.platform === 'gemini' ||
    (props.account?.platform === 'antigravity' && props.account?.type === 'apikey')
})

// OpenAI 生图：模型名以 gpt-image- 开头
const supportsOpenAIImageTest = computed(() => {
  const modelID = selectedModelId.value.toLowerCase()
  if (!modelID.startsWith('gpt-image-')) return false
  return props.account?.platform === 'openai'
})

const supportsImageTest = computed(() =>
  supportsGeminiImageTest.value || supportsOpenAIImageTest.value)
```

> **注意**：前端这个判定只用于"要不要显示提示词框 / 把提示词带上"。真正决定走不走生图路径的是**后端**（后端也按模型名判定）。移植时两端逻辑要保持一致。

### 3.3 核心：startTest 发请求并手动解析 SSE

这是最关键的一段。因为浏览器原生 `EventSource` **只支持 GET**，而这里要用 POST 传 body，所以作者用 `fetch` + `ReadableStream` 手动解析 SSE：

```ts
const startTest = async () => {
  if (!props.account || !selectedModelId.value) return

  resetState()
  status.value = 'connecting'
  abortController = new AbortController()

  const url = `/api/v1/admin/accounts/${props.account.id}/test`
  const response = await fetch(url, {
    method: 'POST',
    headers: {
      Authorization: `Bearer ${localStorage.getItem('auth_token')}`,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      model_id: selectedModelId.value,
      prompt: supportsImageTest.value ? testPrompt.value.trim() : ''
    }),
    signal: abortController.signal
  })

  if (!response.ok) throw new Error(`HTTP error! status: ${response.status}`)

  const reader = response.body?.getReader()
  const decoder = new TextDecoder()
  let buffer = ''

  while (true) {
    const { done, value } = await reader!.read()
    if (done) break
    buffer += decoder.decode(value, { stream: true })
    const lines = buffer.split('\n')
    buffer = lines.pop() || ''        // 最后一行可能不完整，留到下一轮

    for (const line of lines) {
      if (line.startsWith('data: ')) {
        const jsonStr = line.slice(6).trim()
        if (jsonStr) {
          const event = JSON.parse(jsonStr)
          handleEvent(event)          // 分发处理
        }
      }
    }
  }
}
```

### 3.4 事件分发 handleEvent

后端每条 SSE 是一个 JSON 对象，前端按 `type` 分发。生图关键在 `image` 事件：

```ts
const handleEvent = (event: {
  type: string
  text?: string
  model?: string
  success?: boolean
  error?: string
  image_url?: string   // data:image/png;base64,xxxx 形式
  mime_type?: string
}) => {
  switch (event.type) {
    case 'test_start':
      // 连接建立，显示"使用模型 xxx / 正在发送生图请求"
      break

    case 'content':
      // 文本增量（生图时可能是 revised_prompt 或模型的文字说明）
      if (event.text) streamingContent.value += event.text
      break

    case 'image':
      // ★ 生图核心：拿到一张图
      if (event.image_url) {
        generatedImages.value.push({
          url: event.image_url,       // 直接就是 data URI，可直接塞进 <img :src>
          mimeType: event.mime_type
        })
      }
      break

    case 'test_complete':
      status.value = event.success ? 'success' : 'error'
      break

    case 'error':
      status.value = 'error'
      errorMessage.value = event.error || 'Unknown error'
      break
  }
}
```

图片渲染就是普通 `<img>`，因为 `image_url` 本身是 base64 data URI：

```vue
<img :src="image.url" :alt="`test-image-${index + 1}`" />
```

### 3.5 SSE 事件协议（前后端约定，务必照搬）

| type | 字段 | 含义 |
|------|------|------|
| `test_start` | `model` | 开始测试，回带最终使用的模型名 |
| `content` | `text` | 文本增量（生图时可能是 revised_prompt / 说明文字） |
| `image` | `image_url`, `mime_type` | **一张图片**。`image_url` 是 `data:<mime>;base64,<数据>` |
| `test_complete` | `success` | 结束。`success=true` 成功 |
| `error` | `error` | 出错，附错误信息 |

每条消息线格式固定为：`data: {json}\n\n`

## 五、Gemini 生图路径（重点，最易移植）

### 5.1 判断是否为生图模型

`isImageGenerationModel`（位于 `antigravity_gateway_service.go`）决定请求体是否走生图格式：

```go
func isImageGenerationModel(model string) bool {
    m := strings.TrimPrefix(strings.ToLower(model), "models/")
    return m == "gemini-3.1-flash-image" ||
        m == "gemini-3.1-flash-image-preview" ||
        strings.HasPrefix(m, "gemini-3.1-flash-image-") ||
        m == "gemini-3-pro-image" ||
        m == "gemini-3-pro-image-preview" ||
        strings.HasPrefix(m, "gemini-3-pro-image-") ||
        m == "gemini-2.5-flash-image" ||
        m == "gemini-2.5-flash-image-preview" ||
        strings.HasPrefix(m, "gemini-2.5-flash-image-")
}
```

### 5.2 构造生图请求体（关键差异点）

`createGeminiTestPayload`：生图模型必须带 `responseModalities: ["TEXT","IMAGE"]`，否则模型不会返回图片。

```go
// 生图 payload —— 注意 generationConfig
payload := map[string]any{
    "contents": []map[string]any{{
        "role":  "user",
        "parts": []map[string]any{{"text": imagePrompt}},
    }},
    "generationConfig": map[string]any{
        "responseModalities": []string{"TEXT", "IMAGE"}, // ★ 必须
        "imageConfig": map[string]any{
            "aspectRatio": "1:1",                         // 宽高比
        },
    },
}
// 默认提示词（prompt 为空时）：
// "Generate a cute orange cat astronaut sticker on a clean pastel background."
```

对比普通文本 payload（不带 `responseModalities`，带 `systemInstruction`）。这是生图与文本请求唯一的结构差异。

### 5.3 发起请求（API Key 模式，最简单）

```go
// baseURL 默认 https://generativelanguage.googleapis.com
fullURL := fmt.Sprintf(
    "%s/v1beta/models/%s:streamGenerateContent?alt=sse",
    strings.TrimRight(baseURL, "/"), modelID,
)
req, _ := http.NewRequestWithContext(ctx, "POST", fullURL, bytes.NewReader(payload))
req.Header.Set("Content-Type", "application/json")
req.Header.Set("x-goog-api-key", apiKey)   // ★ API Key 放这个头
```

要点：
- 端点用 `:streamGenerateContent?alt=sse`（流式 SSE），也可用 `:generateContent`（一次性 JSON）。
- 鉴权头是 `x-goog-api-key`，不是 `Authorization`。
- Base URL：AI Studio 为 `https://generativelanguage.googleapis.com`。

### 5.4 解析 Gemini 流、提取图片（核心）

`processGeminiStream` 逐行读 SSE，从 `candidates[].content.parts[]` 中提取两类数据：

```go
for _, part := range parts {
    partMap := part.(map[string]any)
    // 1) 文本
    if text, ok := partMap["text"].(string); ok && text != "" {
        sendEvent(TestEvent{Type: "content", Text: text})
    }
    // 2) 图片（内联 base64）
    if inlineData, ok := partMap["inlineData"].(map[string]any); ok {
        mimeType, _ := inlineData["mimeType"].(string) // e.g. "image/png"
        data, _ := inlineData["data"].(string)         // base64 字符串
        if strings.HasPrefix(strings.ToLower(mimeType), "image/") && data != "" {
            sendEvent(TestEvent{
                Type:     "image",
                ImageURL: fmt.Sprintf("data:%s;base64,%s", mimeType, data), // ★ 拼 data URI
                MimeType: mimeType,
            })
        }
    }
}
```

关键点：
- 图片藏在 `parts[].inlineData` 里，字段是 `mimeType` + `data`（base64）。
- 后端把它拼成 `data:image/png;base64,xxxx` 的 Data URI，前端 `<img :src>` 可直接渲染，无需再下载。
- 两种响应包裹格式都要兼容：AI Studio 直接是 `{"candidates":[...]}`；Gemini CLI 包一层 `{"response":{"candidates":[...]}}`。
- 收到 `finishReason` 非空 → 发 `test_complete`。

## 6. 后端实现 — OpenAI 生图路径

判定：模型名以 `gpt-image-` 开头且账号 `platform === 'openai'`（`isOpenAIImageModel`）。

OpenAI 有两条子路径，**移植首选 API Key 路径**（简单、无 OAuth）。

### 6.1 API Key 路径（`testOpenAIImageAPIKey`，推荐移植）

与 Gemini 不同，这是**一次性非流式**请求，不是 SSE。

```go

    "model":           modelID,        // gpt-image-1 / gpt-image-2 ...
    "prompt":          prompt,
    "n":               1,
    "response_format": "b64_json",     // ★ 要 base64，方便直接渲染
}

req.Header.Set("Content-Type", "application/json")
req.Header.Set("Authorization", "Bearer "+apiKey)

// 读取完整响应体（非流式）
// 响应结构：{"data":[{"b64_json":"...","revised_prompt":"..."}]}
for _, item := range result.Data {
    if item.RevisedPrompt != "" {
        sendEvent(TestEvent{Type: "content", Text: item.RevisedPrompt})
    }
    if item.B64JSON != "" {

            Type:     "image",
            ImageURL: "data:image/png;base64," + item.B64JSON,

sendEvent(TestEvent{Type: "test_complete", Success: true})
```

要点：
- 端点 `POST /v1/images/generations`，`response_format: "b64_json"`。
- 响应是普通 JSON（非 SSE），`data[].b64_json` 即图片，`revised_prompt` 是模型改写后的提示词。
- 后端同样拼成 `data:image/png;base64,...` 再通过 `image` 事件下发，与 Gemini 对前端完全一致。

### 6.2 OAuth 路径（`testOpenAIImageOAuth`，移植可选/跳过）

走 ChatGPT 内部 Codex `/responses` 接口（`https://chatgpt.com/backend-api/codex/responses`），把生图当作 tool 调用，用 `Bearer access_token` + `chatgpt-account-id` 头，响应用 `collectOpenAIImagesFromResponsesBody` 解析。依赖 OAuth 令牌与内部协议，**VSCode 插件移植时建议直接跳过，只做 API Key 路径**。

## 7. 迁移到 VSCode 插件（Vue3 + TS + Node）

### 7.1 关键架构差异

| 维度 | 原项目 | VSCode 插件 |
|------|--------|-------------|
| UI 层 | 浏览器里的 Vue3 页面 | Webview 里的 Vue3 页面（同样是浏览器环境） |
| 网络发起方 | Go 后端服务器 | 插件的 **Extension Host（Node 进程）** |
| UI↔逻辑通信 | HTTP + SSE（fetch 流式读取） | `webview.postMessage` / `acquireVsCodeApi().postMessage` |
| 跨域 | 后端转发，无跨域 | **必须由 Node 端请求**，Webview 直连第三方 API 会被 CSP/CORS 挡住 |
| 密钥存放 | 服务端数据库 | VSCode `SecretStorage`（不要放 Webview） |

**核心结论**：原项目"Go 后端"这一层，在插件里由 **Extension Host（Node）** 承担。Webview 只负责界面和收图，所有对 Gemini/OpenAI 的真实请求都放在 Node 端。这样既避开 Webview 的 CSP 限制，也不会把 API Key 暴露到前端。

### 7.2 消息协议（取代原来的 SSE）

原项目 UI 通过读取 HTTP 流拿到 `test_start/content/image/...` 事件。插件里改成 Node 通过 `postMessage` 把**同样语义**的事件推给 Webview。事件结构可直接复用：

```ts
// shared/types.ts —— Webview 与 Node 共用

  model?: string
  success?: boolean
  error?: string

  mime_type?: string

// Webview -> Node：发起生图
export interface StartGenerateMessage {
  command: 'startGenerate'
  requestId: string
  platform: 'gemini' | 'openai'

  prompt: string
```

### 7.3 Node 端：Gemini 生图实现

在 Extension Host 里新建一个服务模块。Node 18+ 自带 `fetch` 与 `ReadableStream`，可以直接复刻原项目的 SSE 流式读取逻辑。

```ts
// src/services/geminiImage.ts
import type { TestEvent } from '../shared/types'

const AI_STUDIO_BASE = 'https://generativelanguage.googleapis.com'

// 复刻 isImageGenerationModel：判断是否走生图路径
export function isGeminiImageModel(model: string): boolean {
  const m = model.toLowerCase().replace(/^models\//, '')
  return (
    m === 'gemini-3.1-flash-image' ||
    m.startsWith('gemini-3.1-flash-image-') ||
    m === 'gemini-3-pro-image' ||
    m.startsWith('gemini-3-pro-image-') ||
    m === 'gemini-2.5-flash-image' ||
    m.startsWith('gemini-2.5-flash-image-')
  )
}

// 复刻 createGeminiTestPayload 的生图分支
function buildGeminiImagePayload(prompt: string) {
  return {
    contents: [{ role: 'user', parts: [{ text: prompt }] }],
    generationConfig: {
      responseModalities: ['TEXT', 'IMAGE'],   // 关键：必须声明 IMAGE
      imageConfig: { aspectRatio: '1:1' }
    }
  }
}

export async function generateGeminiImage(
  apiKey: string,
  modelId: string,
  prompt: string,
  emit: (e: TestEvent) => void,
  signal?: AbortSignal
): Promise<void> {
  const url =
    `${AI_STUDIO_BASE}/v1beta/models/${modelId}:streamGenerateContent?alt=sse`

  emit({ type: 'test_start', model: modelId })

  const resp = await fetch(url, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'x-goog-api-key': apiKey        // AI Studio API Key 鉴权头
    },
    body: JSON.stringify(buildGeminiImagePayload(prompt)),
    signal
  })

  if (!resp.ok || !resp.body) {
    const errText = await resp.text().catch(() => '')
    emit({ type: 'error', error: `API ${resp.status}: ${errText}` })
    return
  }

  await parseGeminiStream(resp.body, emit)   // 见 7.4
}
```

### 7.4 Node 端：Gemini SSE 流解析

这段直接对应原项目 `processGeminiStream`。逐行读取 `data: ` 前缀的 SSE，从 `candidates[].content.parts[]` 中提取 `text` 与 `inlineData`（base64 图片）。

```ts
// src/services/geminiImage.ts (续)
import type { TestEvent } from '../shared/types'

async function parseGeminiStream(
  body: ReadableStream<Uint8Array>,
  emit: (e: TestEvent) => void
): Promise<void> {
  const reader = body.getReader()
  const decoder = new TextDecoder()
  let buffer = ''

  while (true) {
    const { done, value } = await reader.read()
    if (done) break
    buffer += decoder.decode(value, { stream: true })

    const lines = buffer.split('\n')
    buffer = lines.pop() || ''      // 半行留到下一轮

    for (const raw of lines) {
      const line = raw.trim()
      if (!line.startsWith('data: ')) continue
      const jsonStr = line.slice(6).trim()
      if (jsonStr === '[DONE]') { emit({ type: 'test_complete', success: true }); return }

      let data: any
      try { data = JSON.parse(jsonStr) } catch { continue }

      // 兼容 Gemini CLI 格式：{"response": {...}}
      if (data.response) data = data.response

      const candidate = data?.candidates?.[0]
      const parts = candidate?.content?.parts
      if (Array.isArray(parts)) {
        for (const part of parts) {
          if (typeof part.text === 'string' && part.text) {
            emit({ type: 'content', text: part.text })
          }
          const inline = part.inlineData
          if (inline?.mimeType?.toLowerCase().startsWith('image/') && inline.data) {
            emit({
              type: 'image',
              image_url: `data:${inline.mimeType};base64,${inline.data}`,
              mime_type: inline.mimeType
            })
          }
        }
      }

      if (candidate?.finishReason) { emit({ type: 'test_complete', success: true }); return }
      if (data?.error?.message) { emit({ type: 'error', error: data.error.message }); return }
    }
  }
  emit({ type: 'test_complete', success: true })
}
```

### 7.5 Node 端：OpenAI 生图实现（API Key 路径）

对应原项目 `testOpenAIImageAPIKey`。这是**非流式**请求：一次 POST，拿到完整 JSON，从 `data[].b64_json` 取图。

```ts
// src/services/openaiImage.ts
import type { TestEvent } from '../shared/types'

export interface OpenAIImageOptions {
  apiKey: string
  baseUrl?: string            // 默认 https://api.openai.com
  model: string               // gpt-image-1 / gpt-image-2 ...
  prompt: string
  proxyDispatcher?: any
}

const DEFAULT_OPENAI_IMAGE_PROMPT =
  'Generate a cute orange cat astronaut sticker on a clean pastel background.'

export async function generateOpenAIImage(
  opts: OpenAIImageOptions,
  emit: (e: TestEvent) => void
): Promise<void> {
  const base = (opts.baseUrl || 'https://api.openai.com').replace(/\/+$/, '')
  // 简化：若 base 已含 /v1 则不再追加。原项目 buildOpenAIImagesURL 有更完整的规范化逻辑
  const url = /\/v1(\/|$)/.test(base)
    ? `${base}/images/generations`
    : `${base}/v1/images/generations`

  const prompt = opts.prompt.trim() || DEFAULT_OPENAI_IMAGE_PROMPT
  emit({ type: 'test_start', model: opts.model })

  const resp = await fetch(url, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      Authorization: `Bearer ${opts.apiKey}`
    },
    body: JSON.stringify({
      model: opts.model,
      prompt,
      n: 1,
      response_format: 'b64_json'   // 关键：要 base64 而非 URL
    }),
    dispatcher: opts.proxyDispatcher
  } as any)

  const text = await resp.text()
  if (!resp.ok) { emit({ type: 'error', error: `API ${resp.status}: ${text}` }); return }

  let parsed: { data?: Array<{ b64_json?: string; revised_prompt?: string }> }
  try { parsed = JSON.parse(text) } catch { emit({ type: 'error', error: '响应解析失败' }); return }

  const items = parsed.data ?? []
  if (items.length === 0) { emit({ type: 'error', error: 'No images returned' }); return }

  for (const item of items) {
    if (item.revised_prompt) emit({ type: 'content', text: item.revised_prompt })
    if (item.b64_json) {
      emit({
        type: 'image',
        image_url: `data:image/png;base64,${item.b64_json}`,
        mime_type: 'image/png'
      })
    }
  }
  emit({ type: 'test_complete', success: true })
}
```

> OAuth（ChatGPT Codex `/responses`）路径复杂度高（需构造 image_generation tool、解析 `response.output` 中的图片项、处理 `originator`/`chatgpt-account-id` 头），插件场景一般用不到，建议只做 API Key 路径。若确需，参考原项目 `testOpenAIImageOAuth` + `collectOpenAIImagesFromResponsesBody`。

### 7.6 Extension Host：消息路由（替代 Go 的 handler + 路由分发）

Extension 主进程接收 webview 的 `postMessage`，根据 `platform`/`model` 分发到上面两个 service，再把 `emit` 出来的事件回传给 webview。这一层等价于原项目的 `Test` handler + `TestAccountConnection` 的平台分发。

```ts
// src/extension.ts（节选：注册 webview + 消息处理）
import * as vscode from 'vscode'
import { generateGeminiImage } from './services/geminiImage'
import { generateOpenAIImage } from './services/openaiImage'
import type { TestEvent } from './shared/types'

function isGeminiImageModel(model: string): boolean {
  const m = model.toLowerCase().replace(/^models\//, '')
  return (
    m === 'gemini-3.1-flash-image' || m === 'gemini-3.1-flash-image-preview' ||
    m.startsWith('gemini-3.1-flash-image-') ||
    m === 'gemini-3-pro-image' || m === 'gemini-3-pro-image-preview' ||
    m.startsWith('gemini-3-pro-image-') ||
    m === 'gemini-2.5-flash-image' || m === 'gemini-2.5-flash-image-preview' ||
    m.startsWith('gemini-2.5-flash-image-')
  )
}
function isOpenAIImageModel(model: string): boolean {
  return model.toLowerCase().startsWith('gpt-image-')
}

export function registerImagePanel(context: vscode.ExtensionContext, panel: vscode.WebviewPanel) {
  panel.webview.onDidReceiveMessage(async (msg) => {
    if (msg.command !== 'startTest') return
    const { platform, model, prompt, apiKey, baseUrl } = msg.payload

    // emit：把每个事件通过 postMessage 回传（等价于 Go 的 sendEvent 写 SSE）
    const emit = (e: TestEvent) => panel.webview.postMessage({ command: 'testEvent', event: e })

    try {
      if (platform === 'gemini' && isGeminiImageModel(model)) {
        await generateGeminiImage({ apiKey, baseUrl, model, prompt }, emit)
      } else if (platform === 'openai' && isOpenAIImageModel(model)) {
        await generateOpenAIImage({ apiKey, baseUrl, model, prompt }, emit)
      } else {
        emit({ type: 'error', error: `不支持的生图模型: ${model}` })
      }
    } catch (err: any) {
      emit({ type: 'error', error: err?.message ?? String(err) })
    }
  })
}
```

**API Key 存储安全**：不要把 key 放在 webview 或明文配置里。用 VSCode 的 `context.secrets`（SecretStorage）保存，调用时在 extension host 侧读取：

```ts
await context.secrets.store('imageGen.apiKey', key)
const apiKey = await context.secrets.get('imageGen.apiKey')
```

### 7.7 Webview 侧：Vue3 组件消费事件

Webview 里的 Vue 组件基本照搬原 `AccountTestModal.vue`，只把"fetch 读 SSE"换成"postMessage 发起 + onmessage 收事件"。UI 渲染部分（`generatedImages` + `<img :src>`）**完全不变**。

```ts
// src/webview/composables/useImageGen.ts
import { ref } from 'vue'
import type { TestEvent } from '../../shared/types'

const vscode = acquireVsCodeApi()   // VSCode Webview 注入的 API

export function useImageGen() {
  const status = ref<'idle' | 'connecting' | 'success' | 'error'>('idle')
  const generatedImages = ref<{ url: string; mimeType?: string }[]>([])
  const streamingContent = ref('')
  const errorMessage = ref('')

  function startTest(payload: {
    platform: 'gemini' | 'openai'; model: string; prompt: string
  }) {
    status.value = 'connecting'
    generatedImages.value = []
    streamingContent.value = ''
    // 发起：等价于原项目的 fetch POST
    vscode.postMessage({ command: 'startTest', payload })
  }

  // 接收：等价于原项目 handleEvent
  function handleMessage(ev: MessageEvent) {
    const msg = ev.data
    if (msg?.command !== 'testEvent') return
    const event: TestEvent = msg.event
    switch (event.type) {
      case 'content':
        if (event.text) streamingContent.value += event.text
        break
      case 'image':
        if (event.image_url) {
          generatedImages.value.push({ url: event.image_url, mimeType: event.mime_type })
        }
        break
      case 'test_complete':
        status.value = event.success ? 'success' : 'error'
        break
      case 'error':
        status.value = 'error'
        errorMessage.value = event.error || 'Unknown error'
        break
    }
  }

  window.addEventListener('message', handleMessage)
  return { status, generatedImages, streamingContent, errorMessage, startTest }
}
```

模板（与原项目一致，Data URI 可直接渲染）：

```vue
<img v-for="(img, i) in generatedImages" :key="i" :src="img.url" />
```

## 10. 易踩坑清单

1. **Gemini 不出图**：`generationConfig` 少了 `responseModalities:["TEXT","IMAGE"]`。这是最常见原因。
2. **图片不显示**：Webview CSP 没放行 `img-src data:`，或忘了把 base64 拼成 `data:<mime>;base64,<data>`。
3. **鉴权头搞反**：Gemini 用 `x-goog-api-key`，OpenAI 用 `Authorization: Bearer`，两者不通用。
4. **在 Webview 里直接 fetch 第三方 API**：会被 CSP/CORS 拦截，且暴露 API Key。必须放到 Extension Host（Node）。
5. **SSE 分包**：流式读取时一条 `data:` 可能被拆成两个 chunk。务必用 `buffer` 累积、`split('\n')` 后 `pop()` 保留最后半行（Gemini 路径需要；OpenAI API Key 路径是一次性 JSON，无此问题）。
6. **Gemini 响应双格式**：AI Studio 是 `{candidates:[...]}`，Gemini CLI 是 `{response:{candidates:[...]}}`，解析前先 `if (data.response) data = data.response`。
7. **模型判定两端要一致**：Webview（决定是否显示提示词框）与 Node（决定走哪条生图路径）用同一套 `isXxxImageModel` 规则，否则会出现 UI 显示生图框但后端走了文本路径。
8. **Node 版本**：确保 Node 18+（VSCode 内置的 Electron Node 通常满足），才能用全局 `fetch` 与 `ReadableStream`；否则改用 `undici`。
