## 🔄 三、数据流设计

### 3.1 用户消息流

```
用户输入 (Webview)
    ↓ postMessage
Extension: ChatService.sendMessage()
    ↓
Extension: QueryEngine.query()
    ↓ ← → 工具执行循环
    │
    ├→ Tool.execute() (文件/Bash/搜索等)
    │     ↓
    │  VSCode Adapter 适配
    │     ↓
    │  返回结果
    │
    ├→ Provider API 调用
    │     ↓
    │  流式响应
    │     ↓
    │  postMessage 推送到 Webview
    │
    └→ 工具调用决策
          ↓
      继续循环 or 完成
```

### 3.2 new-api 同步流程

```
1. 用户点击「同步」(Webview)
      ↓ postMessage
2. Extension: NewApiSyncService.startSync()
      ↓
3. 启动本地 HTTP 服务器 (localhost:随机端口)
      ↓
4. 打开系统浏览器 → new-api 授权页
      ↓
5. 用户授权
      ↓
6. 浏览器重定向 → http://localhost:<port>/callback?code=xxx
      ↓
7. Extension: 收到 code，调用 new-api exchange 接口
      ↓
8. 获取令牌列表 + 分组模型映射
      ↓
9. postMessage 推送预览数据到 Webview (脱敏)
      ↓
10. 用户勾选并配置模型映射
      ↓ postMessage
11. Extension: 从内存取完整 key，调用 ProviderService.addProvider()
      ↓
12. 写入 ~/.claude/cc-evancod/providers.json
      ↓
13. 完成同步
```

## 🔌 四、通信机制

### 4.1 Extension ↔ Webview 通信协议

#### Message Types (Extension → Webview)

```typescript
type ExtensionMessage = 
  | { type: 'chat.message.stream', data: { content: string, delta: boolean } }
  | { type: 'chat.message.complete', data: { messageId: string } }
  | { type: 'chat.tool.call', data: { toolName: string, args: any } }
  | { type: 'chat.tool.result', data: { toolName: string, result: any } }
  | { type: 'provider.list', data: { providers: Provider[] } }
  | { type: 'provider.active.changed', data: { activeId: string } }
  | { type: 'newapi.sync.preview', data: { tokens: Token[], models: string[] } }
  | { type: 'session.restored', data: { messages: Message[] } }
  | { type: 'cost.update', data: { tokens: number, cost: number } }
  | { type: 'error', data: { message: string, code?: string } }
```

#### Message Types (Webview → Extension)

```typescript
type WebviewMessage =
  | { type: 'chat.send', data: { content: string } }
  | { type: 'chat.stop', data: { messageId: string } }
  | { type: 'provider.create', data: CreateProviderInput }
  | { type: 'provider.update', data: UpdateProviderInput }
  | { type: 'provider.delete', data: { id: string } }
  | { type: 'provider.activate', data: { id: string } }
  | { type: 'provider.test', data: { id: string } }
  | { type: 'newapi.sync.start', data: { siteUrl: string } }
  | { type: 'newapi.sync.import', data: { tokens: string[], mappings: ModelMapping[] } }
  | { type: 'session.load', data: { sessionId: string } }
  | { type: 'ready', data: null }
```

## 🔐 六、安全设计

### 6.1 敏感信息处理

1. **API Key 安全**
   - 存储: 使用 VSCode Secret Storage API
   - 传输: Extension Host 内部处理，不通过 postMessage
   - 展示: Webview 只显示脱敏版本（`sk-****1234`）

2. **OAuth Token 安全**
   - new-api code: 一次性使用，5 分钟 TTL
   - 完整 key: 仅在 Extension 内存中短暂存在
   - 本地回调: 仅监听 localhost，随机端口

3. **文件访问权限**
   - 遵守 VSCode workspace.fs API 权限模型
   - 工作区外文件需用户确认

### 6.2 网络安全

1. **HTTPS 验证**
   - 默认验证 SSL 证书
   - 允许用户配置信任自签名证书（警告提示）

2. **代理支持**
   - HTTP/HTTPS/SOCKS5 代理
   - 读取系统代理设置
   - 支持认证代理

